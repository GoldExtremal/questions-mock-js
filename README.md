# JS Mock Interview

Небольшой статический лендинг для подготовки к собеседованию по JavaScript.  
Проект рассчитан на быстрый просмотр тем, навигацию по разделам, поиск по вопросам и раскрытие ответов прямо на странице.

## Возможности

- Фиксированная верхняя шапка с названием проекта и быстрым поиском.
- Левое фиксированное меню разделов с подсветкой текущего раздела.
- Последовательная прокрутка по разделам сверху вниз без скачков между колонками.
- Поиск по вопросам, задачам, подзаголовкам и кодовым примерам.
- Навигация по совпадениям через стрелки `↑` и `↓`.
- Подсветка найденных вхождений в тексте и в коде.
- Раскрывающиеся вопросы с подробными ответами и уточнениями.
- Блок задач с отдельными практическими заданиями и шаблонами кода.
- Кнопка копирования в кодовых блоках.
- Тёмная визуальная тема, удобная для долгого чтения.

## Как это работает

Проект состоит из трёх основных файлов:

- [`index.html`](/home/goldextremal/Projects/questions-mock-js/index.html) - статическая разметка страницы.
- [`styles.css`](/home/goldextremal/Projects/questions-mock-js/styles.css) - оформление, сетка, тема и адаптация.
- [`app.js`](/home/goldextremal/Projects/questions-mock-js/app.js) - логика поиска, навигации, раскрытия блоков и копирования кода.

Страница не требует сборки и работает как обычный статический сайт.

## Поведение интерфейса

### Навигация

- Клик по разделу в левом меню прокручивает страницу к соответствующему блоку.
- Активный раздел подсвечивается автоматически при скролле.
- Верхняя граница раздела выравнивается по нижней границе шапки.

### Поиск

- Поиск работает по заголовкам, уточняющим пунктам, описаниям и коду.
- Ответы внутри раскрытых панелей не участвуют в поиске.
- При вводе запроса показывается счётчик совпадений и кнопки перехода.
- Совпадения можно пролистывать циклично.
- Если совпадение найдено в свернутом блоке, соответствующий вопрос автоматически раскрывается.

### Ответы и задачи

- У каждого основного вопроса есть кнопка ответа.
- По нажатию открывается подробное пояснение.
- В задачах доступны кодовые шаблоны и краткие подсказки по решению.
- Кнопка копирования меняет состояние после успешного копирования.

## Нюансы И Ограничения

Это полезно учитывать при доработках и ревью проекта.

### История И Навигация

- Навигация по разделам меняет адрес через `pushState`, но для полноценной поддержки кнопок `Back` и `Forward` стоит отдельно проверить синхронизацию состояния.
- Активный раздел подсвечивается во время скролла, поэтому любые изменения высоты шапки или отступов могут потребовать проверки логики прокрутки.

### Контент И Разметка

- Контент сейчас статически вшит в `index.html`, поэтому правки вопросов, ответов и задач нужно делать внимательно, чтобы не рассинхронизировать текст с поиском и подсветкой.
- Поиск опирается на `data-base-text` и `data-raw`, поэтому при добавлении новых элементов важно не забывать про эти атрибуты.
- Раскрывающиеся блоки и подсветка совпадений завязаны на структуру DOM, поэтому крупные изменения разметки лучше проверять вместе с логикой поиска.

### Производительность

- Поиск пересобирает подсветку текста и кода на лету, что удобно для текущего объёма данных, но при заметном росте контента может потребовать оптимизации.
- Для небольшого статического лендинга это нормально, но при расширении проекта лучше заранее подумать о более явной модели данных.

### Стили И Браузерное Поведение

- Тёмная тема и нативные элементы браузера могут вести себя по-разному в зависимости от платформы, особенно у поля поиска и autofill.
- Если меняется высота шапки или ширина сайдбара, стоит отдельно проверять прокрутку к разделам и отображение активного состояния.

### Что Стоит Проверять После Изменений

- Поиск по заголовкам, уточнениям и коду.
- Переходы по стрелкам между совпадениями.
- Автораскрытие вопросов и задач при поиске.
- Кнопку копирования кода.
- Скролл к разделам из бокового меню.

## Как Добавлять Контент

### Новый Вопрос

1. Добавь новый блок вопроса в нужный раздел внутри `index.html`.
2. Укажи основной заголовок вопроса в элементе с `data-base-text`.
3. Если нужны уточнения, добавь их в список уточняющих пунктов.
4. Если у вопроса есть код, помести его в `code[data-raw]`, чтобы работали подсветка и поиск.
5. Если у вопроса должен быть разворачиваемый ответ, добавь его в структуру ответа рядом с кнопкой `Ответ`.

### Новая Задача

1. Добавь новый `practice-item` в блок `Задачи`.
2. Дай ему понятный заголовок и короткое пояснение перед кодом.
3. Для кодового шаблона используй `code[data-raw]`.
4. Если нужен ответ, добавь текст внутри панели ответа, но не дублируй лишнюю структуру.

### Что Важно Не Забыть

- Текст, который должен искать сайт, должен быть отражён в `data-base-text` или `data-raw`.
- Для ответов и пояснений лучше использовать человекочитаемые формулировки, потому что поиск работает по тем же текстам, что и пользователь видит на экране.
- После добавления нового блока обязательно проверить, что он корректно раскрывается, копируется и участвует в поиске.

## Как Устроен Поиск

Поиск в проекте работает не как простой фильтр, а как подсветка и навигация по совпадениям.

### Что Ищется

- Заголовки разделов.
- Заголовки вопросов.
- Уточняющие пункты.
- Кодовые блоки.
- Краткие описания и подписи, если они отмечены как поисковые.

### Что Не Ищется

- Тексты внутри раскрытых панелей ответа.
- Служебная разметка, которая нужна только для отображения интерфейса.

### Как Это Работает

1. Пользователь вводит запрос в поиск.
2. Сайт подсвечивает совпадения в тексте и коде.
3. Показывается счётчик найденных совпадений.
4. Стрелками можно переходить между совпадениями.
5. При необходимости совпадающий вопрос или задача раскрывается автоматически.

### На Что Опирается Поиск

- На `data-base-text` для обычного текста.
- На `code[data-raw]` для кодовых блоков.
- На структуру DOM, чтобы открывать нужный вопрос или задачу при совпадении.

## Безопасные Изменения

Если хочется доработать проект без поломок, лучше соблюдать несколько правил.

### Можно Менять Относительно Безопасно

- Тексты вопросов и ответов, если сохранить нужные `data-base-text` и `data-raw`.
- Стили оформления, если не менять ключевые классы и структуру блоков.
- Порядок и названия разделов в меню, если одновременно обновить соответствующие якоря.

### Нужно Проверять Особенно Внимательно

- Изменение высоты шапки.
- Перестройку структуры вопросов и задач.
- Переименование классов, которые использует `app.js`.
- Любые правки в кодовых блоках.
- Добавление новых уровней вложенности.

### Что Лучше Проверять После Любой Правки

- Скролл к разделам из меню.
- Поиск по заголовкам и коду.
- Переходы по совпадениям.
- Автооткрытие нужного блока.
- Кнопку копирования кода.
